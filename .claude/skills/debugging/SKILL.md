---
name: debugging
description: >
  Aktiviere diesen Skill automatisch wenn: ein Fehler, Bug, Exception,
  Crash oder unerwartetes Verhalten auftritt. Wenn jemand fragt
  "warum funktioniert das nicht" oder "hilf mir debuggen".
---

**4-Phasen Debugging-Methodik:**
1. **Verstehen**: Fehlermeldung vollständig lesen, nicht raten
2. **Reproduzieren**: Minimales Beispiel finden das den Fehler zeigt
3. **Ursache finden**: Root Cause analysieren, nicht Symptome behandeln
4. **Beheben & Verifizieren**: Fix anwenden, testen, sicherstellen dass nichts anderes kaputt geht

**Gotchas-Sektion:**
Häufige Fehler beim Debugging:
- Voreilige Fixes ohne Verständnis
- Symptome statt Ursachen beheben
- Keine Tests zur Fehler-Reproduktion geschrieben
